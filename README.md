# 洛基之影（Shadow of Loki）

**my-neuro** 生态下的 **live-2d 社区插件**，面向“游戏陪玩 / 游戏攻略辅助”场景：结合对话与截图上下文，记录任务、区域、Boss、章节和角色状态，并在需要时优先检索本地攻略库，再从多平台来源补充信息，输出更贴近当前进度的辅助回答。

**仓库地址：** [https://github.com/A-night-owl-Rabbit/my-neuro-plugin-loki-shadow](https://github.com/A-night-owl-Rabbit/my-neuro-plugin-loki-shadow)

---

## 功能概览

| 能力 | 说明 |
|------|------|
| **游戏状态追踪** | 工具 `loki_shadow_track` 记录任务、地图、Boss、章节、角色等上下文 |
| **攻略 / 剧情查询** | 工具 `loki_shadow_query` 发起结构化查询 |
| **五源并行检索** | 游民星空、B站、TapTap、NGA、米游社等来源并行拉取 |
| **本地攻略库优先** | 优先读取仓库内 `game-guides/` 社区共建攻略库 |
| **双模型容灾** | 主 Agent（默认 DeepSeek 系）+ 后备 Agent（默认 Qwen 系） |
| **会话增强** | 结合近期游戏状态优化低质量搜索词 |
| **陪玩向回答** | 偏分步引导，避免硬贴大段原文 |

## 游戏攻略库分享计划

为了让插件越用越强，这个仓库同时维护一个社区共建的游戏攻略库：[`game-guides/`](./game-guides)

### 共享原则

- 攻略请使用 `.txt` 格式上传。
- 一份攻略对应一个文件，文件名尽量写清游戏名、任务名、章节名、Boss 名或主题关键词。
- 欢迎补充主线流程、支线任务、Boss 打法、角色养成、配队思路、版本活动、剧情解析等内容。
- 内容越清晰，插件按文件名筛选时命中率越高。
- 上传之后，你贡献的内容会变成公共攻略库的一部分，帮助后来的使用者。
- 人人为我，我为人人。

### 推荐命名方式

```text
游戏名_任务或章节_主题.txt
游戏名_Boss名_打法攻略.txt
游戏名_角色名_养成配队.txt
```

例如：

```text
鸣潮_曙光停摆于荒地之上_主线攻略.txt
绝区零_第2季第6章_剧情解析.txt
原神_无相之雷_打法攻略.txt
```

## 环境要求

- **Node.js**（与 my-neuro live-2d 一致）
- **依赖**：`axios`、`cheerio`。本仓库已附带 **`node_modules/`** 与 **`package-lock.json`**（与插件目录内 `package.json` 对应），便于离线环境或需锁定依赖版本时直接使用。若你更习惯在 **live-2d** 根目录统一安装依赖，仍可在根目录执行 `npm install axios cheerio`；此时 Node 会按模块解析规则优先使用就近的 `node_modules`。
- **Windows**：窗口检测相关能力依赖 PowerShell

## 适用场景

- 用户正在游玩单机 / 二游 / 剧情游戏时，结合截图或聊天内容进行辅助。
- 用户卡关，希望获得不过度剧透的分步引导。
- 用户想讨论当前章节、任务、Boss 或角色，但不希望被未来剧情直接剧透。
- 用户需要将零碎截图信息转成结构化搜索词进行攻略查询。

## 工作流程

插件核心流程由 [`orchestrator.js`](./orchestrator.js) 编排，大致如下：

1. 检测当前游戏（或使用显式传入的游戏名）
2. 扫描本地攻略库
3. 分析已有内容是否足够回答
4. 必要时从多来源并行抓取补充资料
5. 整合结果并生成可复用攻略内容
6. 结合当前会话状态生成最终回答

## 安装方式

将整个插件目录放到 my-neuro 的社区插件路径下，例如：

```text
live-2d/plugins/community/loki-shadow/
```

目录内应包含 `index.js`、`metadata.json`、`plugin_config.json` 及同仓库中的各模块文件。

**依赖安装（二选一即可）：**

1. **使用本仓库自带的 `node_modules/`**（推荐离线 / 固定版本）：克隆或解压后无需再执行 `npm install`，只要目录结构完整即可。
2. **在 live-2d 根目录安装**（与旧版文档一致）：若未使用自带目录或希望与根目录依赖合并管理，可在 **live-2d** 根目录执行：

```bash
npm install axios cheerio
```

在 my-neuro 中启用插件后重启或按界面提示重新加载插件。

## 配置说明（`plugin_config.json`）

本仓库中的 `plugin_config.json` 为公开模板：`api_key` 等敏感项为空，请勿把填好真实 Key 的文件提交到 Git。

### 主要配置项

| 配置项 | 说明 |
|--------|------|
| `guide_library_path` | 默认使用插件目录下的 `./game-guides`，也可改成你自己的可写目录 |
| `sub_agent` | 主 LLM：API 地址、模型名、温度、`max_tokens` 等 |
| `fallback_agent` | 后备 LLM，字段同上；`api_key` 可与主 Agent 相同 |
| `*_search_limit` / `gamersky_download_limit` | 各来源数量上限，可按网络与配额调整 |
| `max_content_length` | 单段送入模型的最大字符数，防止上下文过长 |

### 编辑 JSON 时的编码

若使用记事本等保存为 UTF-8 带 BOM，可能导致主程序解析 `plugin_config.json` 报错。建议使用 VS Code 等编辑器保存为 UTF-8（无 BOM），或使用已包含 BOM 自动剥离逻辑的较新版本 my-neuro 主程序。

### 攻略库分享计划与配置的关系

- 默认路径已经指向仓库内的公共攻略库，克隆后可直接使用。
- 你也可以把 `guide_library_path` 改到自己的目录，单独维护私有攻略库。
- 如果你愿意把内容共享出来，推荐把整理好的 `.txt` 攻略再提交回本仓库的 `game-guides/`。
- 公共库越大，首次命中本地攻略的概率越高，重复抓取的次数越少。

## 工具说明

### `loki_shadow_track`

上报当前观察到的游戏状态（游戏名、任务、Boss、区域、章节、角色等），用于增强后续查询，不直接搜索攻略。

### `loki_shadow_query`

发起正式攻略 / 剧情类查询。请尽量使用结构化关键词（任务名、区域、Boss 名等），效果优于零碎截图原文。

## 检索性能说明

当前本地攻略库检索逻辑是：

1. 递归扫描 `txt` 文件
2. 按文件名包含游戏名做初筛
3. 把候选文件名交给模型选择最相关文件
4. 读取命中文件内容

这不是全文索引检索，更接近“文件名筛选 + 内容读取”。

以当前仓库内这批攻略为例：

- 攻略文件数：27
- 总体积：约 486 KB
- 全库扫描平均耗时：约 0.9 ms
- 去重索引首次构建耗时：约 9.1 ms
- 最大单文件读取平均耗时：约 0.33 ms

所以按现在这批规模，几乎感觉不到本地 I/O 压力，实际延迟更多来自模型调用和联网抓取。

但当文件量明显增大时，影响会线性上升：

- **约 1,000 个文件**：本地扫描通常仍在几十毫秒级，影响轻微。
- **约 10,000 个文件**：扫描和候选列表会明显变慢，可能进入百毫秒到秒级区间。
- **真正更大的瓶颈**：候选文件名过多时，需要把更多文件名交给模型选择，模型输入会膨胀，响应时间和成本会上升得比本地读盘更明显。

结论：

- 现在这批攻略量不会明显影响检索效率。
- 后续真要做大规模共建，优先保证文件名规范，其次再考虑轻量索引或按游戏分目录。
